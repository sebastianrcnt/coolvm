## cool8 ISA

### 기본 파라미터

| 항목            | 값                             |
| --------------- | ------------------------------ |
| 데이터 폭       | 8비트                          |
| 주소 공간       | 8비트 논리 주소 (은행당 256B)  |
| 명령어 길이     | 8비트 고정                     |
| 레지스터        | 4개 (r0~r3), 범용, r0은 항상 0 |
| 프로그램 카운터 | 8비트                          |
| 플래그          | 없음                           |
| 물리 메모리     | 뱅크 선택으로 최대 64KiB       |

- 레지스터를 4개로 잡은 이유가 핵심인데요 — 레지스터 지정에 2비트만 쓰면 되니까, 8비트 명령어 안에서 오퍼랜드 2~3개를 넣을 여유가 생겨요. 8개로 하면 3비트씩 먹어서 즉시값(immediate)을 넣을 공간이 없어지거든요.

### 명령어 인코딩

8비트 안에서 이렇게 쪼갭니다:

**R-type (레지스터 연산):**

```
[opcode:4][rd:2][rs:2]
```

**I-type (즉시값):**

```
[opcode:4][rd:2][imm:2]
```

**B-type (분기):**

```
[opcode:4][rs:2][offset:2]
```

### 명령어 세트

```
Opcode  이름     형식     동작
──────────────────────────────────────────
0x0     ADD     R       rd = rd + rs
0x1     SUB     R       rd = rd - rs
0x2     AND     R       rd = rd & rs
0x3     OR      R       rd = rd | rs
0x4     NOR     R       rd = ~(rd | rs)     ← NOT은 NOR rd, r0으로
0x5     LDI     I       rd = imm (0~3)      ← 작은 상수 로드
0x6     ADDI    I       rd = rd + imm
0x7     SHL     I       rd = rd << imm
0x8     SHR     I       rd = rd >> imm
0x9     LD      R       rd = MEM[rs]
0xA     ST      R       MEM[rd] = rs
0xB     BEZ     B       if rs == 0: PC += sext(off)
0xC     BNZ     B       if rs != 0: PC += sext(off)
0xD     -       -       예약됨
0xE     JAL     R       rd = PC+1; PC = rs  ← 함수 호출/점프
0xF     SYS     -       시스템 콜 (halt 등)
```

### 설계 트레이드오프 포인트

**"왜 MOV가 없어요?"** — `ADD rd, r0`으로 되거든요. r0이 항상 0이니까요. 이렇게 명령어 하나를 아끼는 게 8비트 ISA에서는 생존 전략이에요.
**"LDI가 0~3밖에 안 돼요?"** — 맞아요, 즉시값이 2비트라서요. 큰 상수를 만들려면 이런 식으로 조합해야 해요:

```asm
LDI  r1, 3      ; r1 = 0x03
SHL  r1, 2      ; r1 = 0x0C
ADDI r1, 2      ; r1 = 0x0E
```

불편하지만, 이게 진짜 실제 RISC 설계에서 겪는 문제예요. MIPS의 LUI+ORI, RISC-V의 LUI+ADDI가 정확히 같은 이유로 존재하거든요.

**"분기 범위가 너무 좁지 않아?"** — BEZ/BNZ는 2비트 오프셋만 써서 `pc+1` 기준으로 `-2..+1` 바이트만 점프해요. 너무 짧은 대가를 치르는 대신 플래그를 지우고 숨겨진 상태를 없애는 편의 선택입니다.

### 어셈블러 디렉티브

`cool8` 어셈블러는 지금부터 다음 지시어를 지원합니다.

```
.byte  value[, value...]
```

- 즉시값 또는 라벨 주소를 바이트 단위로 데이터로 삽입합니다.
- 각 값은 하위 8비트만 기록됩니다.
- 예: `.byte 0x41, 65, 10`

```
.ascii  "string"
```

- 문자열 리터럴을 바이트 시퀀스로 삽입합니다.
- `\n`, `\t`, `\r`, `\0`, `\\`, `\"` 이스케이프를 지원합니다.
- 예: `.ascii "A\\n"`

라벨은 로컬 라벨 `.loop` 처럼 점 표기법으로도 사용할 수 있으며, 해당 주소가 즉시 쓰여집니다.

### 메모리 맵

```
0x00 ┌──────────────┐
     │  코드          │  프로그램 시작
     │  ...          │
0xE0 ├──────────────┤
     │  스택 (↓)     │  SP 초기값 = 0xDF
0xF0 ├──────────────┤
     │  I/O (MMIO)   │  기본 UART 출력: ST addr(0xF0), value
0xFE ├──────────────┤
     │  I/O (MMIO)   │  확장 MMIO
0xFF ├──────────────┤
     │ BANK_SELECT   │  읽기/쓰기 시 현재 뱅크 선택
0xFF └──────────────┘
```

256바이트 논리 주소 공간은 동일하게 유지됩니다. 다만 `BANK_SELECT`를 통해 활성 뱅크를 바꾸면
실제 물리 주소는 `bank << 8 | addr(0x00~0xFF)`로 해석됩니다.

`BANK_SELECT` 값도 8비트라 뱅크는 `0x00..0xFF`이므로, 총 64KiB까지 확장됩니다.

### 뱅크 스위칭 사용법

- `BANK_SELECT` MMIO 주소: `0xFF`
- 주소 접근 규칙:
  - `addr < 0xF0`: 은행 스위칭이 적용되는 RAM/ROM
  - `addr >= 0xF0`: MMIO(기존 장치)
- 동작 예시

```asm
; r2에 0xFF(=BANK_SELECT) 주소를 준비했다고 가정
LDI r1, 3
ADDI r1, -2
SHL r1, 3
LDI r3, 3
SHL r3, 2
ST  r2, r1      ; bank=3 선택

; 이후 LD/ST는 r2/r3로 지정한 주소가 bank 3의 물리 공간을 가리킴
LD  r0, r3
```

주의: ST로 `BANK_SELECT`를 바꾼 직후 바로 이어지는 코드 fetch도 새 뱅크에서 계속 진행됩니다.
따라서 코드가 여러 은행으로 나뉘는 프로그램은 뱅크 전환 위치를 의도적으로 설계해야 합니다.
