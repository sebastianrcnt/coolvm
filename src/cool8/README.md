## cool8 ISA

### 기본 파라미터

| 항목            | 값                             |
| --------------- | ------------------------------ |
| 데이터 폭       | 8비트                          |
| 주소 공간       | 8비트 (256B)                   |
| 명령어 길이     | 8비트 고정                     |
| 레지스터        | 4개 (r0~r3), 범용, r0은 항상 0 |
| 프로그램 카운터 | 8비트                          |
| 플래그          | Zero (Z), Carry (C)            |

- 레지스터를 4개로 잡은 이유가 핵심인데요 — 레지스터 지정에 2비트만 쓰면 되니까, 8비트 명령어 안에서 오퍼랜드 2~3개를 넣을 여유가 생겨요. 8개로 하면 3비트씩 먹어서 즉시값(immediate)을 넣을 공간이 없어지거든요.

### 명령어 인코딩

8비트 안에서 이렇게 쪼갭니다:

**R-type (레지스터 연산):**

```
[opcode:4][rd:2][rs:2]
```

**I-type (즉시값):**

```
[opcode:4][rd:2][imm:2]
```

**B-type (분기):**

```
[opcode:4][offset:4]
```

### 명령어 세트

```
Opcode  이름     형식     동작
──────────────────────────────────────────
0x0     ADD     R       rd = rd + rs
0x1     SUB     R       rd = rd - rs
0x2     AND     R       rd = rd & rs
0x3     OR      R       rd = rd | rs
0x4     NOR     R       rd = ~(rd | rs)     ← NOT은 NOR rd, r0으로
0x5     LDI     I       rd = imm (0~3)      ← 작은 상수 로드
0x6     ADDI    I       rd = rd + imm
0x7     SHL     I       rd = rd << imm
0x8     SHR     I       rd = rd >> imm
0x9     LD      R       rd = MEM[rs]
0xA     ST      R       MEM[rd] = rs
0xB     BEQ     B       if Z: PC += sext(off)
0xC     BNE     B       if !Z: PC += sext(off)
0xD     BCS     B       if C: PC += sext(off)
0xE     JAL     R       rd = PC+1; PC = rs  ← 함수 호출/점프
0xF     SYS     -       시스템 콜 (halt 등)
```

### 설계 트레이드오프 포인트

**"왜 MOV가 없어요?"** — `ADD rd, r0`으로 되거든요. r0이 항상 0이니까요. 이렇게 명령어 하나를 아끼는 게 8비트 ISA에서는 생존 전략이에요.
**"LDI가 0~3밖에 안 돼요?"** — 맞아요, 즉시값이 2비트라서요. 큰 상수를 만들려면 이런 식으로 조합해야 해요:

```asm
LDI  r1, 3      ; r1 = 0x03
SHL  r1, 2      ; r1 = 0x0C
ADDI r1, 2      ; r1 = 0x0E
```

불편하지만, 이게 진짜 실제 RISC 설계에서 겪는 문제예요. MIPS의 LUI+ORI, RISC-V의 LUI+ADDI가 정확히 같은 이유로 존재하거든요.

**"분기 범위가 너무 좁지 않아?"** — B-type의 offset이 4비트(부호 확장하면 -8~+7)라서, PC 기준 앞뒤 8바이트밖에 못 뛰어요. 멀리 가려면 JAL로 간접 점프해야 하고요. 이것도 실제 아키텍처에서 short branch / long jump 구분이 왜 존재하는지를 체감하게 해줘요.

### 메모리 맵

```
0x00 ┌──────────────┐
     │  코드          │  프로그램 시작
     │  ...          │
0xE0 ├──────────────┤
     │  스택 (↓)     │  SP 초기값 = 0xDF
0xF0 ├──────────────┤
     │  I/O (MMIO)   │  UART 등
0xFF └──────────────┘
```

256바이트에서 코드 224바이트, 스택 약 16바이트, I/O 16바이트. 빡빡하지만 교육용으로 핵심을 보기엔 충분해요.
